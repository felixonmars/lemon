Pucker-Strength Unit Testing

Lemon is a Unit Testing framework that forces the developer into a class-to-case method-to-unit mirrored construction of tests. This promotes the very proper technique for unit testing, as it helps ensure test coverage.

Let's say our library consists of the class X with method #a:

  class X
    def a; "a"; end
  end

We write a testcase as follows:

  testcase X do
    unit :a do
      X.new.a.assert.is_a? String
    end
  end

The run the lemon test from the commandline:

  $ lemon test/cases/*.rb

There are numerous configuration options:

  USAGE: lemon [options] tests1 [tests2 ...]

  OPTIONS:
       --prefix                   path prefix of target environment
       --bindir                   directory for commands
       --libdir                   directory for libraries
       --datadir                  directory for shared data
       --mandir                   directory for man pages
       --docdir                   Directory for documentation
       --sysconfdir               directory for system configuration files
       --localstatedir            directory for local state data
       --libruby                  directory for ruby libraries
       --librubyver               directory for standard ruby libraries
       --librubyverarch           directory for standard ruby extensions
       --siteruby                 directory for version-independent aux ruby libraries
       --siterubyver              directory for aux ruby libraries
       --siterubyverarch          directory for aux ruby binaries
       --rbdir                    directory for ruby scripts
       --sodir                    directory for ruby extentions
       --rubypath                 path to set to #! line
       --rubyprog                 ruby program used for installation
       --makeprog                 make program to compile ruby extentions
       --extconfopt               options to pass-thru to extconf.rb
       --withoutext               do not compile/install ruby extentions
       --withoutdoc               do not generate documentation
       --shebang                  shebang line (#!) editing mode (all,ruby,never)
       --doctemplate              document template to use (html|xml)
       --testrunner               Runner to use for testing (auto|console|tk|gtk|gtk2)
       --installdirs              install location mode (std,site,home :: libruby,site_ruby,$HOME)

  COMMON OPTIONS:
    -q --quiet                    Silence output
       --verbose                  Provide verbose output
    -n --no-write                 Do not write to disk
